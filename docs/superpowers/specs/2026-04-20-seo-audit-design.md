# SEO診断設計: 整体院ひざこぞう

## 目的

整体院ひざこぞうのサイト全体を、ローカルファイルと公開サイトの情報から多角的に診断し、今後のSEO改善で最初に取り組むべき事項を優先順位化する。

優先順位は次の順で扱う。

1. 全体弱点の洗い出し
2. ローカルSEO / MEO
3. 検索流入の増加
4. 問い合わせ導線の改善

## 前提

- アクセス表記は「柏駅西口 徒歩8分」に統一する。
- 最優先テーマは「膝痛」とする。
- 最初の診断ではSearch ConsoleやGoogleビジネスプロフィールの実データは使わない。
- まずはコード変更ではなく、改善優先順位表とSEOマップを作る。

## 診断対象

- トップページ
- 膝痛関連の症状ページ
  - 変形性膝関節症
  - 膝の水
  - 膝の外側の痛み
  - 膝の裏側の痛み
  - 鵞足炎
- 膝痛関連ブログ記事
- sitemap.xml
- robots.txt
- title / meta description / canonical
- 構造化データ
- 内部リンク
- LINE、電話、症状ページへの導線
- 誇張表現、医療広告的に危うい表現、信頼性に影響する表現

## 評価軸

### P0: 信頼性と整合性

院情報、アクセス表記、電話番号、営業時間、構造化データ、強すぎる改善表現を確認する。ローカルSEOと信頼性の土台になるため、最初に扱う。

### P1: 膝痛SEOの設計

膝痛系ページの役割分担、検索意図の重複、記事から症状ページへの内部リンク、主力ページへの評価集約を確認する。

### P2: 検索流入の改善

title、meta description、見出し、FAQ、記事タイトル、ロングテールキーワードとの一致を確認する。

### P3: 問い合わせ導線

問い合わせ改善は今回の優先順位では最後だが、明らかに機会損失がある導線は診断に含める。

## 成果物

1. SEO改善優先順位表
   - 優先度、領域、問題、対象ページ、影響、対応案をまとめる。
2. 膝痛SEOマップ
   - 主要検索意図と担当ページを整理する。
3. 30日アクションプラン
   - すぐ直す項目、リライト項目、新規記事候補を分ける。

## 完了条件

- P0 / P1 / P2 / P3の優先順位が明確になっている。
- 膝痛を中心に、どのページを主力化するかが分かる。
- 次にコード変更する場合の作業範囲が明確になっている。
