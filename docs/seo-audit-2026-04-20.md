# 整体院ひざこぞう SEO診断レポート

作成日: 2026-04-20  
対象: https://hizakozou.jp  
前提: Search Console / Googleビジネスプロフィールの実データは未使用。ローカルファイル、公開サイト、公式ドキュメントをもとにした初期診断。

## 結論

今すぐやるべきことは、記事追加ではなく **P0の整合性と信頼性の修正** です。

サイトの基礎SEOはかなり整っています。59URLがsitemapに載り、主要59ページでtitle、description、canonical、H1は欠けていません。トップ、症状ページ、記事ページには構造化データも入っています。

一方で、ローカルSEOと信頼性に効く情報の統一がまだ弱いです。特に「柏駅西口 徒歩8分」と「徒歩9分」の混在、症状ページでの「根本改善」など強い訴求、膝痛ページ群の役割分担の曖昧さが、最初に直すべきポイントです。

優先順位は次の順です。

1. P0: 表記統一、強い訴求の調整、ローカル情報の整合
2. P1: 膝痛SEOのページ設計と内部リンク整理
3. P2: title / description / FAQ / 記事リライト
4. P3: 問い合わせ導線の細部改善

## P0: 最初に直すべき項目

| 優先 | 領域 | 問題 | 対象 | 影響 | 最初の対応 |
|---|---|---|---|---|---|
| P0 | ローカルSEO | 「徒歩8分」と「徒歩9分」が混在 | トップ、全症状ページ | Googleビジネスプロフィール、ユーザー認識、NAP整合性に影響 | 全ページを「柏駅西口 徒歩8分」に統一 |
| P0 | 信頼性 | 「根本改善」が85件あり、一部症状ページでは強く見える | 症状ページ、記事タイトル/本文 | 医療・健康系サイトとして過度な期待を生みやすい | 「改善を目指す」「サポート」「負担を減らす」へ調整 |
| P0 | 表現リスク | 「手術回避」「回復期間を大きく短縮」「必ず原因」など断定寄り表現がある | 股関節、五十肩、膝水、膝裏など | 信頼性低下、広告表現上のリスク | 効果保証に見える文を弱め、受診目安を併記 |
| P0 | LocalBusiness | トップのMedicalBusiness情報は入っているが、GBPとの一致確認が未実施 | `index.html` | ローカルSEOの土台 | GBPの住所、電話、営業時間、URL、カテゴリ、写真と照合 |
| P0 | URL整理 | `blog.html` / `blog-detail.html` はnoindex誘導ページとして残存 | 旧ブログ導線 | 大きな問題ではないが、移行意図の明文化が必要 | sitemap非掲載、noindex維持、内部リンクが新URLを向いているか確認 |

### なぜP0か

Googleビジネスプロフィールの公式ヘルプでは、ローカル検索は主に **関連性、距離、知名度** に基づくと説明されています。距離や関連性の理解を助けるには、院名、住所、電話、営業時間、アクセス表記を揃えることが重要です。

また、厚生労働省の医療広告関連資料では、誇大な広告や、比較優良に見える表現が問題になり得るとされています。整体院サイトは医療機関そのものとは扱いが異なる場合がありますが、健康・身体に関わる訴求なので、SEO上も「誠実で過度に断定しない表現」に寄せる方が安全です。

## P1: 膝痛SEOの設計

膝痛を最優先にする場合、今の資産は十分あります。ただし、ページの役割を整理しないと「どのページを上げたいのか」がぼやけます。

### 推奨する親子構造

| 検索意図 | 主力ページ | 支える記事 |
|---|---|---|
| 柏市で膝痛を相談したい | トップページ、`symptoms/knee-osteoarthritis.html` | 膝は治らないと思っていませんか、施術で確認する7つのポイント |
| 変形性膝関節症 | `symptoms/knee-osteoarthritis.html` | 歩き始め、階段、下腿外旋、スウェイバック、膝痛日常動作 |
| 膝に水がたまる | `symptoms/knee-effusion.html` | 膝に水が溜まる原因と対処法、膝裏痛、歩き始め |
| 膝の内側が痛い | `symptoms/pes-anserine-bursitis.html` | 鵞足炎、伏在神経、下腿外旋、7つのポイント |
| 膝の外側が痛い | `symptoms/knee-lateral-pain.html` | 腸脛靭帯症候群、スウェイバック、膝裏痛、下腿外旋 |
| 膝の裏側が痛い | `symptoms/knee-posterior-pain.html` | ベーカー嚢腫、下腿外旋、鵞足炎 |
| 階段・歩き始め・立ち上がり | 記事群から主力症状ページへ送る | 階段、歩くと膝、歩き始め、日常動作 |

### 重要な判断

「膝痛」という大きい語はトップページと変形性膝関節症ページで受けます。  
ブログ記事はそれぞれロングテールを拾い、最終的に5つの膝症状ページへ内部リンクで集約します。

今は関連リンクの仕組みがあり、症状ページから記事への導線もあります。次に強めたいのは逆方向、つまり **記事本文の中から主力症状ページへ文脈付きでリンクすること** です。

例:

- 階段記事 → 変形性膝関節症、膝の水
- 歩き始め記事 → 変形性膝関節症
- 膝の水記事 → 膝関節水腫ページ
- 鵞足炎記事 → 鵞足炎ページ
- 膝外側記事 → 膝外側の痛みページ

## P2: 検索流入を増やすための改善

### title / description

全ページに設定済みなので、欠落修正ではなくCTR改善が中心です。

優先リライト候補:

| ページ | 現状 | 改善方向 |
|---|---|---|
| `symptoms/knee-osteoarthritis.html` | 「変形性膝関節症・膝痛の相談」 | 「柏市で変形性膝関節症・歩き始め/階段の膝痛相談」まで検索意図を広げる |
| `symptoms/knee-effusion.html` | 「膝の水・関節水腫を根本改善」 | 「膝に水がたまる・腫れが続く方へ」に寄せる |
| `symptoms/pes-anserine-bursitis.html` | 「膝の内側の痛み・鵞足炎を根本改善」 | 「膝の内側が痛い・鵞足炎の相談」に寄せる |
| `blog/index.html` | ブログ一覧 | `BlogPosting`や`CollectionPage` JSON-LD追加はP2で検討 |

### FAQ

記事ページと症状ページにFAQは入っています。次の改善は、FAQを検索意図別に分けることです。

膝痛系FAQの優先テーマ:

- 病院へ行くべき膝痛の目安
- 膝に水があるとき運動してよいか
- 変形性膝関節症で歩いてよいか
- 階段で痛いとき避けたい動き
- 膝の内側と外側で見方は違うか

### 新規記事より先にリライト

既に膝痛カテゴリ14本があります。新規記事を足す前に、以下のリライトが先です。

1. 「階段で膝が痛むときに考えたいこと」を主力記事化
2. 「歩き始めの膝の痛み」を変形性膝関節症ページへ集約
3. 「膝の痛みが気になるときの日常動作」をトップ/症状ページの補助記事化
4. 「膝は治らないと思っていませんか」は表現を少し落としつつ、来院判断記事として育てる

## P3: 問い合わせ導線

優先順位は最後ですが、明らかな機会損失だけ拾います。

| 項目 | 現状 | 改善案 |
|---|---|---|
| 記事中CTA | 記事にLINE CTAあり | 膝痛記事では「膝痛の相談」文脈に揃える |
| 症状ページCTA | CTAあり | 「柏駅西口 徒歩8分」「初回の進め方」への不安解消を近くに置く |
| 電話/LINE | トップにあり | 症状ページ下部で電話番号、LINE、営業時間を統一表示 |
| GBP連携 | 未確認 | GBPの予約/電話/ルート検索との整合を次フェーズで確認 |

## 30日アクションプラン

### 1週目: P0修正

1. 全ページの「徒歩9分」を「徒歩8分」に統一する。
2. 症状ページtitle / descriptionの「根本改善」を弱める。
3. 「手術回避」「回復期間を大きく短縮」「必ず原因」などを、保証に見えない表現へ置き換える。
4. トップのMedicalBusiness構造化データをGBPと照合する。
5. `blog.html` / `blog-detail.html` が内部リンクで使われていないか確認し、旧導線として維持するならコメントや運用メモに残す。

### 2週目: 膝痛SEOマップの実装

1. 膝痛系5症状ページを主力ページとして定義する。
2. 膝痛記事14本を、どの症状ページへ送るか決める。
3. 記事本文の自然な箇所に、症状ページへの文脈リンクを追加する。
4. トップページの「膝痛タイプから探す」を、5症状ページへより明確に接続する。

### 3週目: 主力記事リライト

1. 階段記事を「階段 膝 痛い 柏市」系の主力記事にする。
2. 歩き始め記事を「歩き始め 膝 痛い」系の主力記事にする。
3. 膝の水記事を、医療機関受診目安と整体で見る範囲を明確にして補強する。
4. 鵞足炎、膝外側、膝裏の記事を症状ページと対になる記事として整える。

### 4週目: 計測準備

1. Search Consoleで以下を確認する。
   - `柏市 膝痛`
   - `柏 膝 整体`
   - `変形性膝関節症 柏`
   - `階段 膝 痛い`
   - `膝 水がたまる`
   - `膝 内側 痛い`
   - `膝 外側 痛い`
2. Googleビジネスプロフィールで以下を確認する。
   - 表示回数
   - 電話
   - ルート検索
   - ウェブサイトクリック
   - 口コミ数と返信状況
   - 写真の更新状況
3. 実データをもとに、次の30日分の新規記事/リライト計画を決める。

## 次にコード変更するなら

最初の実装は小さく切るのが良いです。

1. `徒歩9分` を `徒歩8分` に統一
2. 膝痛症状ページ5本のtitle / descriptionを調整
3. 強い表現を一括で弱める
4. 膝痛記事から症状ページへの内部リンクを追加
5. 生成対象のブログ記事は `content/source/*.md` を編集し、必ず `npm run generate:blog` を実行
